# 项目理念

这个项目关注的是个人 Agent 系统在成为产品之前应该是什么样子。

核心想法是：有用的 LLM Agent 系统不只是模型和工具的组合，也是一组工作习惯：什么时候请求帮助，什么时候保持本地执行，什么时候花更多模型预算，什么时候放慢，以及什么时候把最终决定留给人。

## 设计价值观

- **诚实：** 说清楚已经存在什么、什么只是设想、什么还没有做。
- **用户控制：** 对敏感或有后果的动作，保持用户控制。
- **本地意识：** 把本地部署当作有隐私和可靠性含义的设计选择。
- **运行上的克制：** 偏向保守表述，而不是夸张但无法验证的说法。
- **系统思维：** 把模型行为和路由、成本、安全、可靠性连接起来。

## 希望实现的 Agent 使用体验

这组文档所指向的 Agent 使用体验，应该是有用但不过度假装自治。

它应该：

- 解释重要权衡。
- 保持任务范围清晰。
- 在风险升高时请求确认。
- 谨慎使用工具。
- 偏向可逆步骤。
- 把失败当作信息，而不是需要隐藏的东西。

## 为什么这件事重要

个人 LLM 系统往往从实验开始。如果没有文档，配置会漂移，模型选择会变得随意，安全假设也会停留在隐含状态。

这个仓库的价值在于把这些假设显性化。它的意义是文档纪律，而不是声称系统已经完整。
